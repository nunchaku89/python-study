## Goal

* 데이터를 처음 받았을 때 확인해야 하는 항목을 이해한다.
* EDA(탐색적 데이터 분석)의 기본 절차를 익힌다.
* 데이터 품질을 빠르게 파악하는 방법을 학습한다.

## Implementation

파일: `exercises/eda.py`

구현 내용

* `shape`를 이용하여 데이터 크기를 확인했다.
* `columns`와 `dtypes`를 확인했다.
* `info()`를 통해 데이터 타입과 결측치를 확인했다.
* `describe()`를 이용하여 수치형 데이터의 기본 통계를 확인했다.
* `head()`와 `tail()`로 실제 데이터를 확인했다.
* `isna().sum()`으로 컬럼별 결측치를 확인했다.

## 핵심 학습

EDA는 데이터를 분석하기 전에 **데이터를 이해하는 과정**이다.

데이터를 바로 분석하기보다 다음 순서를 먼저 확인한다.

1. 데이터 크기
2. 컬럼 구조
3. 데이터 타입
4. 결측치
5. 기본 통계
6. 실제 데이터 형태

## Learned

* 분석 전에 데이터의 품질을 먼저 확인해야 한다.
* 결측치를 확인하고 처리 전략을 세워야 한다.
* `describe()`를 통해 데이터의 전체적인 분포를 빠르게 파악할 수 있다.
* EDA는 모든 데이터 분석 프로젝트의 시작점이다.

## 실무 활용

새로운 CSV, Excel, 데이터베이스 테이블을 받으면 가장 먼저 수행하는 작업이다.

EDA를 통해

* 잘못된 데이터
* 결측치
* 이상치
* 데이터 타입 오류

를 조기에 발견할 수 있다.
