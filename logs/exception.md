# Exception Handling

## 학습일

* 2026-07-08

## Goal

* 예외 처리의 필요성을 이해한다.
* `try` / `except`를 사용하여 프로그램이 비정상 종료되지 않도록 구현한다.
* 잘못된 입력과 파일 접근 오류를 안전하게 처리하는 방법을 익힌다.
* 데이터 분석에서 예외 처리가 왜 중요한지 이해한다.

## Implementation

파일: `exercises/exception.py`

구현 내용

* 사용자 입력을 숫자로 변환하고, 잘못된 입력에 대해 `ValueError`를 처리했다.
* 존재하지 않는 파일을 읽을 때 `FileNotFoundError`를 처리했다.
* 문자열 리스트에서 숫자가 아닌 데이터를 제외하고 평균을 계산했다.
* 학생 정보(List of Dictionary)에서 점수가 숫자인 학생만 평균 계산에 포함했다.

핵심 학습

* `try` 블록에서는 정상적으로 실행할 코드를 작성하고, 예외가 발생하면 `except` 블록에서 처리하도록 구현했다.
* 평균 계산 시 잘못된 데이터를 제외하기 위해 유효한 데이터 개수를 함께 관리했다.
* 예외를 단순히 무시하는 것이 아니라, 어떤 데이터가 제외되었는지 출력하여 처리 과정을 확인했다.

## Learned

* 예외 처리는 프로그램이 종료되지 않고 계속 실행될 수 있도록 도와준다.
* `ValueError`와 `FileNotFoundError`처럼 발생 가능한 예외를 구체적으로 처리하는 것이 좋다.
* 데이터 분석에서는 잘못된 데이터를 제외하거나 정제하는 과정이 매우 중요하다.
* Python 내장 함수 이름(`sum` 등)을 변수명으로 사용하지 않는 것이 좋으며, `total`과 같은 의미 있는 변수명을 사용하는 것이 권장된다.

## 실무 활용

* CSV나 Excel 데이터에는 숫자가 아닌 값, 누락된 값, 잘못된 형식의 데이터가 포함되는 경우가 많다.
* 예외 처리는 이러한 데이터를 안전하게 처리하기 위한 기초이며, 이후 Pandas의 결측치 처리(`NaN`), 데이터 타입 변환, 데이터 정제(Data Cleaning)와 자연스럽게 연결된다.
* 데이터 품질이 일정하지 않은 환경에서도 프로그램이 안정적으로 동작하도록 만드는 중요한 기술이다.
